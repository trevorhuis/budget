Architecture Overview

Design principle: No external KV store (Redis). All optimizations work within the existing Postgres-only stack.
Request PathBeforeAfterImprovement
Hot path (cached cookie) 1–2 DB queries 0 DB queries Eliminated entirely
Warm path (cache miss) 1–2 DB queries 1 indexed query Consistent, fast
Session writes Every request Once per day >80% reduction
Rate limits In-memory only Persisted in DB Survives restart

Design Decisions

DecisionChoiceRationale
cookieCache.maxAge 300s (5 min) Balance of performance and security. Default compact strategy (implicit).
Session lifetime 7 days / 1 day refresh Standard sliding window. Reduces DB write amplification.
Rate limiting 60s / 30 req, DB storage Production-only. Shared across instances. No Redis needed.
ID generation uuidv7() Time-sortable. Keeps package (used in business layer). Single config object.
Connection pool Defaults unchanged Cookie cache eliminates hot-path pressure. No tuning needed.
Session cleanup setInterval every 6h Simple, idempotent. Skipped in test environment.

4. src/lib/session-cleanup.ts — Background cleanup

📄 api/src/lib/session-cleanup.ts NEW
import { db } from "../db/database.js";

const CLEANUP_INTERVAL_MS = 6 * 60 * 60 * 1000; // 6 hours

export const startSessionCleanup = () => {
  if (process.env.NODE_ENV === "test") {
    return;
  }

  const runCleanup = async () => {
    try {
      const result = await db
        .deleteFrom("authSessions")
        .where("expiresAt", "<", new Date())
        .executeTakeFirst();

      const deletedCount = Number(result.numDeletedRows ?? 0);

      if (deletedCount > 0) {
        console.log(
          `[session-cleanup] Removed ${deletedCount} expired session(s)`,
        );
      }
    } catch (error) {
      console.error("[session-cleanup] Failed to clean up expired sessions:",
        error instanceof Error ? error.message : String(error),
      );
    }
  };

  void runCleanup();
  setInterval(runCleanup, CLEANUP_INTERVAL_MS);
};
Impact: Prevents unbounded growth of authSessions. Runs immediately on startup, then every 6 hours. Safely skipped in test environment to avoid test pollution.

Operational Notes

Emergency session invalidation: Bump session.cookieCache.version in auth.ts to force all users to re-authenticate. No Redis flush needed.
Per-user invalidation: Delete the user's row(s) from authSessions. On next request, the cookie cache falls through to DB, finds nothing, and rejects.
Monitoring: Watch authSessions table size. Cleanup runs every 6 hours. If growth is abnormal, check expiresAt values or reduce session.expiresIn.
